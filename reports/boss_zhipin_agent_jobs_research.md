## 二、核心岗位职责（高频出现）

### 2.1 Agent系统设计与开发
- 负责智能Agent系统的设计、开发与迭代
- 实现核心逻辑、任务规划、工具调用、多模态交互等模块
- 设计并实现Agent关键模块：多轮对话状态管理、任务自动拆解与规划(Task Planning)、工具调用策略(Tool Use)
- 多Agent协作机制（角色分工/冲突解决）及结果评估优化
- 构建Agent与外部系统的高效交互层（API/数据库/第三方工具调用）

### 2.2 RAG系统开发
- 负责大模型RAG和Agent框架的开发和升级
- 构建高效的混合检索系统，结合语义搜索和关键词搜索
- 设计并开发AI Agent和Memory记忆数据召回机制
- RAG优化（检索增强生成）

### 2.3 大模型工程化
- 基于大语言模型(LLM)进行Agent能力扩展
- 完成记忆机制、自主学习、多Agent协作等功能的研发
- LLM选型、微调(Fine-tuning)与推理优化
- Prompt Engineering（Few-shot/CoT/ReAct提示策略）
- 对接外部工具与API，实现Agent的工具调用与任务执行闭环

### 2.4 系统架构与工程
- 负责AI相关API接口的设计与开发，确保系统稳定性和可扩展性
- 设计可扩展的Agent系统架构，确保高并发与分布式处理能力
- 持续优化RAG系统性能指标：召回率、准确率和响应时间
- 全栈开发：前端界面与后端服务、数据库设计与优化

### 6.2 rag-pro项目描述模板

```
【Modular RAG MCP Server | L3生产级RAG系统】
- 设计并实现模块化RAG检索流水线，支持Embedding检索→重排序→LLM生成的完整链路
- 实现混合检索(Hybrid Search)：语义搜索+BM25关键词搜索融合，提升召回率
- 构建MCP Server协议接口，使RAG能力可被Claude/Cursor等客户端原生调用
- 设计3层测试体系(单元/集成/E2E)，确保系统稳定性与可维护性
- 支持多向量数据库(Chroma/Qdrant)切换，实现检索层可插拔架构
- 技术栈: Python, FastAPI, LangChain, Chroma, SentenceTransformers
```

### 6.3 Langresearch项目描述模板

```
【LangResearch | 基于LangGraph的智能研究Agent】
- 基于LangGraph构建8阶段研究Agent工作流：需求澄清→任务规划→子任务路由→
  并行研究子图→综合合成→报告生成→质量验证
- 实现Evaluator-Optimizer模式：报告 draft→质量评估→自动重写，最多2轮优化
- 设计HITL人机协作节点：当LLM对用户query置信度<0.65时自动中断，引导澄清
- 嵌套子图架构：每个子任务独立运行search→think→finalize循环，支持预算控制
- 技术栈: Python, LangGraph, LangChain, Tavily, OpenAI API
```

## 八、总结

### 市场趋势
1. **Agent开发工程师岗位正在爆发式增长**，2024-2025年需求激增，薪资水平明显高于传统后端
2. **LangChain/LangGraph已成为事实标准**，几乎所有岗位都要求或加分
3. **RAG是Agent的基础能力**，不掌握RAG的Agent开发者竞争力大减
4. **全栈能力越来越重要**，能独立搭建完整Agent系统的候选人更受青睐
5. **多Agent协作是高级岗位的区分点**，单Agent已趋于同质化

### 当前竞争力评估
- **rag-pro**: 已具备生产级RAG系统的工程能力，在"RAG开发"子赛道有竞争力
- **Langresearch**: 架构设计能力突出(Evaluator-Optimizer、HITL、嵌套子图)，但工程完整度不足
- **组合优势**: 两个项目互补（被动检索 vs 主动研究），这是独特的差异化优势

### 关键建议
1. **2周内补齐前端界面 + 多Agent协作**，这是简历通过筛选的门槛
2. **将"MCP协议"加入技术栈**，这是2025-2026年的新兴热门技能
3. **量化项目成果**：不要说"实现了RAG"，要说"召回率从X提升到Y"
4. **准备技术演讲**：能清晰解释Evaluator-Optimizer和HITL设计的人很少，这是面试加分项
