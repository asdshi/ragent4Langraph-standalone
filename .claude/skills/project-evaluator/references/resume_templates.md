# 简历话术模板 (Resume Templates)

> 针对 RAG/Agent 项目的简历描述，按岗位分类  
> 原则：**量化 > 描述，动作 > 参与，结果 > 过程**

---

## 核心公式

### 开发岗 STAR-量化公式
```
【项目名】
- 背景(Situation): [业务痛点，用数字说明规模]
- 技术(Technology): [具体技术栈，按层次列出]
- 优化(Action): [你做了什么，不是团队做了什么]
- 成果(Result): [必须有量化数字，至少2个指标]
```

### 算法岗 PRIDE 公式
```
【项目名】
- 问题(Problem): [基线指标，现有方法的缺陷]
- 方法(Research): [你的创新策略/算法]
- 实验(Experiment): [对比数据，消融实验结果]
- 产出(Impact): [论文/专利/开源影响力]
```

## 一、开发岗简历模板

### 模板 D1: 企业级 RAG 知识库系统

```markdown
【企业级对话式知识库系统】
- 背景: 解决多用户文档隔离与实时问答需求，支持日均 500+ 查询
- 技术: FastAPI + Vue3 + LangGraph + Chroma + PostgreSQL + Redis
        6 阶段异步 Ingestion Pipeline，混合检索（Dense+Sparse+RRF）
- 优化: 滑动窗口记忆压缩降低 70% Token 消耗；并发信号量控制防止 API 配额打爆；
        用户中断后脏 checkpoint 自动回滚，保证数据一致性
- 成果: 支持 10+ 文档格式端到端处理，P99 延迟 < 2s，
        MCP 协议接入可被外部 Agent 生态调用
```

### 模板 D2: Agent 自动化系统

```markdown
【Agent 驱动的 RPA 自动化平台】
- 背景: 客服部门日均 5000+ 重复工单，人力成本高，响应慢
- 技术: LangGraph + Mem0 + WebShaper + 20+ Tool 集成
        多 Agent 协同（规划 Agent + 执行 Agent + 审核 Agent）
- 优化: 异常重试机制使工具调用成功率从 70%→95%；
        异步并发处理，吞吐量提升 5 倍；
        语义缓存降低 60% LLM API 调用成本
- 成果: 自动化率 80%，效率提升 3 倍，节省人力成本 200 万/年
```

### 模板 D3: Multi-Agent 协作系统

```markdown
【Multi-Agent 智能协作平台】
- 背景: 复杂任务需要多角色协同，单 Agent 无法覆盖全流程
- 技术: AutoGen/CrewAI + Supervisor 模式 + 消息队列 + 角色权限系统
        3 个专业 Agent（需求分析 + 预算规划 + 行程执行）
- 优化: 层级式通信降低 40% 冗余交互；共识机制解决 Agent 冲突；
        动态任务分配根据负载自动调度
- 成果: 任务完成率从 65% 提升至 92%，平均处理时长缩短 55%
```

### 模板 D4: 高可用检索系统

```markdown
【高可用混合检索引擎】
- 背景: 单一向量检索召回率低（62%），无法满足企业知识库精度要求
- 技术: Milvus + BM25 + RRF 融合 + Cross-Encoder Reranker + Redis 缓存
        查询重写（HyDE）+ 子查询拆分 + 意图识别
- 优化: 混合检索使召回率从 62%→85%；Reranker 使 Top-3 准确率提升 18%；
        语义缓存降低 70% 重复查询成本
- 成果: P99 检索延迟 2s→300ms，服务 1000+ 员工，满意度 95%
```

## 二、算法岗简历模板

### 模板 A1: Agentic RAG 策略优化

```markdown
【Agentic RAG 自主检索策略优化】
- 问题: 传统 RAG 在多跳推理场景召回率仅 62%，缺乏动态决策能力
- 方法: 基于 ReAct 框架设计自主规划检索策略，引入多跳推理路径搜索；
        设计自适应检索深度控制器，根据中间结果动态调整召回范围；
        提出改进的 RRF 融合权重，结合语义相关性与文档新鲜度
- 实验: 在 HotPotQA 数据集上 F1 提升 12%，对比 5 种 baseline；
        消融实验: 规划策略贡献 8%，路径排序贡献 4%，自适应控制贡献 3%
- 产出: 论文在投 EMNLP（一作），代码开源 300+ Stars
```

### 模板 A2: 记忆压缩算法

```markdown
【Agent 记忆压缩与长期记忆机制】
- 问题: 长对话场景下历史消息线性增长，上下文窗口利用率低，成本高昂
- 方法: 设计分层记忆架构（STM/MTM/LTM），滑动窗口自动压缩；
        基于摘要的重要性评分算法，关键信息保留率 >90%；
        跨会话长期记忆检索，支持时间衰减与关联召回
- 实验: 存储开销降低 60%，关键信息召回率 92%；
        对比全量历史: 回答准确率仅下降 2%，Token 成本降低 75%
- 产出: 开源组件被 3 个项目引用，技术博客阅读量 10万+
```

### 模板 A3: Reranker 微调

```markdown
【领域自适应 Reranker 微调】
- 问题: 通用 Cross-Encoder 在垂直领域排序效果差，NDCG@10 仅 0.58
- 方法: 基于 LoRA 微调 BERT-large 作为领域 Reranker；
        构造 10万+ 领域对比样本对（相关/不相关/弱相关三级标注）；
        设计难负样本挖掘策略，提升模型判别能力
- 实验: NDCG@10 从 0.58→0.76，MRR 提升 31%；
        对比通用模型: 领域查询提升显著，通用查询无退化；
        推理延迟 < 50ms，满足在线 serving 要求
- 产出: 模型开源 HuggingFace，月下载量 2000+
```

### 模板 A4: Multi-Agent 协作策略

```markdown
【Multi-Agent 动态协作策略设计】
- 问题: 固定角色分配的 Multi-Agent 系统存在负载不均、通信冗余问题
- 方法: 提出基于任务复杂度的动态角色分配算法；
        设计消息重要性评分机制，过滤低价值通信；
        引入共识投票机制解决 Agent 间冲突
- 实验: 在旅行规划 Benchmark 上，任务完成率从 65%→92%；
        平均通信轮次减少 40%，冲突解决成功率 96%；
        消融实验: 动态分配贡献 18%，消息过滤贡献 12%
- 产出: 开源框架获 150+ Stars，被 2 个企业项目采用
```

## 五、避坑指南

### ❌ 不要写的描述
```markdown
- "参与开发了一个 AI 客服系统" → 太模糊，无个人贡献
- "使用 LangChain 搭建了 RAG 系统" → 调包侠既视感
- "提升了系统性能" → 无量化，不可信
- "学习了 Transformer 架构" → 这是学习过程，不是项目成果
- "负责后端开发" → 没有体现具体技术决策
```

### ✅ 要写的描述
```markdown
- "基于 LangGraph + MCP 构建多 Agent 客服系统，工具调用成功率 94%"
- "设计混合检索策略（Dense+Sparse+RRF），召回率从 62%→85%"
- "响应时长从 3.2s 降至 0.8s，日处理 10万+ 对话"
- "提出基于 RL 的子图采样算法，F1 提升 12%，对比 5 种 baseline"
- "搭建异步 Ingestion Pipeline，6 阶段处理，支持 10+ 文档格式"
```

### 动词替换表
| ❌ 弱动词 | ✅ 强动词 |
|:---|:---|
| 参与 | 独立负责 / 主导 |
| 使用 | 基于...设计 / 选型并落地 |
| 搭建 | 端到端实现 / 从零构建 |
| 优化 | 将...从 X 降至 Y / 提升 Z% |
| 学习 | 掌握并应用 / 手撕实现 |
| 做了 | 提出 / 设计 / 实现 / 验证 |
