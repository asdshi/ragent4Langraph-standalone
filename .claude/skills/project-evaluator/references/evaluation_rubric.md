# 五维度评分细则 (Evaluation Rubric)

> 每个子项按 1-5 分评分，5=优秀, 4=良好, 3=及格, 2=薄弱, 1=缺失  
> 维度总分 = 子项平均分 × 维度权重系数，最终换算为百分制

---

## A. 工程架构与代码质量（权重 30%）

### A1 架构分层（1-5分）

| 分数 | 标准 | 检查点 |
|:---:|:---|:---|
| 1 | 无架构 | 单文件脚本，所有逻辑耦合 |
| 2 | 基础分层 | 有 model/service/controller 分层，但边界模糊，循环依赖 |
| 3 | 清晰模块化 | 各模块职责单一，接口定义清晰，依赖关系合理 |
| 4 | 领域驱动 | 按业务域拆分（如 Ingestion/Retrieval/Generation 独立），支持插件扩展 |
| 5 | 企业级架构 | 水平扩展、服务降级、熔断、多租户、事件驱动、CQRS |

**加分信号**:
- [ ] 工厂模式管理多 Provider（LLM/Embedding/Reranker）
- [ ] 配置化管理，不同环境（dev/test/prod）可切换
- [ ] 有清晰的 Pipeline/Workflow 编排（LangGraph/Airflow/自研）
- [ ] 存储层分离（向量库、关系库、缓存各司其职）

### A2 异步与并发（1-5分）

| 分数 | 标准 |
|:---:|:---|
| 1 | 全同步阻塞，文件上传阻塞主线程 |
| 2 | 简单异步（有 `async def` 但未做并发控制） |
| 3 | 有并发控制（信号量、连接池、限流） |
| 4 | 完整异步链路 + 背压机制 + 任务队列 |
| 5 | 以上全部 + 队列削峰 + 分布式任务调度 |

**加分信号**:
- [ ] 文件上传后台异步处理，前端轮询状态
- [ ] 全局并发控制防止 API 配额被打爆
- [ ] 耗时操作使用任务队列（Celery/RQ/自研）
- [ ] 流式输出（SSE/WebSocket）而非阻塞等待

### A3 错误处理与容错（1-5分）

| 分数 | 标准 |
|:---:|:---|
| 1 | 无 try-catch，出错直接 500 |
| 2 | 基础异常捕获，统一错误返回 |
| 3 | 分级容错（网络超时重试、模型降级、默认兜底） |
| 4 | 事务级一致性（脏数据清理、状态回滚） |
| 5 | 以上全部 + 熔断器 + 监控告警 + 自动恢复 |

**加分信号**:
- [ ] LLM 调用失败有降级策略（切换模型/返回提示）
- [ ] 向量化失败标记状态，支持断点续传
- [ ] 用户中途取消生成，优雅终止并清理资源
- [ ] 文件处理失败不影响其他文件

### A4 代码规范（1-5分）

| 分数 | 标准 |
|:---:|:---|
| 1 | 无类型注解、无文档字符串、命名混乱 |
| 2 | 基础规范（PEP8、类型注解覆盖 >50%） |
| 3 | 良好规范 + Docstring + 静态检查（mypy/ruff） |
| 4 | 有代码审查配置（pre-commit、CI lint） |
| 5 | 架构决策记录（ADR）+ 接口契约测试 + 代码覆盖率报告 |

### A5 测试覆盖（1-5分）

| 分数 | 标准 |
|:---:|:---|
| 1 | 无任何测试 |
| 2 | 少量单元测试，覆盖率 <40% |
| 3 | 单元测试覆盖核心模块（>60%） |
| 4 | 三层测试（Unit + Integration + E2E） |
| 5 | 以上全部 + 契约测试 + 性能基准测试 + 混沌测试 |

**维度 A 换算**:
```
A_score = (A1 + A2 + A3 + A4 + A5) / 5 * 6  →  满分 30
```

## B. 算法深度与技术创新（权重 25%）

### B1 检索策略（1-5分）

| 分数 | 策略层级 | 代表能力 |
|:---:|:---|:---|
| 1 | 基础 RAG | 直接向量检索 + 简单 Prompt |
| 2 | 初级优化 | 基础分块 + 简单过滤 |
| 3 | 高级 RAG | 混合检索（Dense+Sparse）、Rerank、查询重写 |
| 4 | Modular RAG | 路由决策、HyDE、多路召回、Self-RAG |
| 5 | 前沿 RAG | GraphRAG、Agentic RAG、多模态 RAG、自研索引算法 |

**加分信号**:
- [ ] 实现查询重写（Query Rewriting/Expansion）
- [ ] 有检索结果后处理（去重、重排序、截断策略）
- [ ] 实现多跳推理（Multi-hop Retrieval）
- [ ] 支持上下文压缩（Context Compression/RAG-Fusion）

### B2 查询优化（1-5分）

| 分数 | 标准 |
|:---:|:---|
| 1 | 无查询处理，原始输入直送检索 |
| 2 | 简单关键词提取或停用词过滤 |
| 3 | 查询重写/扩展（Synonym/HyDE） |
| 4 | 指代消解 + 子查询拆分 + 意图识别 |
| 5 | 自适应查询策略（根据问题类型动态选择检索方式） |

### B3 上下文管理（1-5分）

| 分数 | 标准 |
|:---:|:---|
| 1 | 无记忆，或简单拼接历史 |
| 2 | 固定窗口历史（最近 N 轮） |
| 3 | 滑动窗口 + 摘要压缩 |
| 4 | 多级记忆（STM/MTM/LTM） |
| 5 | 记忆个性化（用户画像、长期偏好学习、记忆衰减机制） |

**加分信号**:
- [ ] 滑动窗口自动压缩历史消息
- [ ] 滚动摘要 + MySQL/PostgreSQL 归档
- [ ] LTM 长期记忆，跨会话召回
- [ ] Checkpoint Timeline 可回溯

### B4 自研算法/策略（1-5分）

| 分数 | 标准 |
|:---:|:---|
| 1 | 纯调包，无任何自研逻辑 |
| 2 | 自研 Prompt 模板或简单后处理规则 |
| 3 | 自研分块策略、自定义召回逻辑、改进的 RRF 权重 |
| 4 | 自研轻量级模型（LoRA 微调 Reranker/Embedding） |
| 5 | 发表论文级创新（新架构、新损失函数、新训练范式） |

### B5 模型训练与微调（1-5分）

| 分数 | 标准 |
|:---:|:---|
| 1 | 仅调用第三方 API |
| 2 | 使用 LoRA/QLoRA 微调过模型 |
| 3 | 完整训练流程（数据构造→训练→评估→部署） |
| 4 | 强化学习优化（PPO/DPO/GRPO 对齐 Agent 行为） |
| 5 | 自研训练框架或新对齐算法 |

**维度 B 换算**:
```
B_score = (B1 + B2 + B3 + B4 + B5) / 5 * 5  →  满分 25
```

## 总分计算

```
Total = A_score + B_score + C_score + D_score + E_score
      = (A1+A2+A3+A4+A5)/5*6 + (B1+B2+B3+B4+B5)/5*5 + (C1+C2+C3+C4+C5)/5*4 + (D1+D2+D3+D4+D5)/5*3 + (E1+E2+E3)/3*3.33
```

满分 100 分。
